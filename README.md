## Overview

This repository contains structured, module-by-module summaries of every course in the [Self-Taught MBA roadmap](https://roadmap.sh/r/mba-self-taught) — a comprehensive, self-directed curriculum covering the core disciplines of business education: finance, economics, strategy, operations, marketing, human resources, analytics, project management, supply chain, entrepreneurship, and more.

Each summary follows a consistent, easy-to-scan format:

- Core idea of each module
- Key concepts and definitions
- Main arguments and insights
- Frameworks, models, and tools
- Formulas and equations (where applicable)
- Practical implications for real-world application
- Learning outcomes
- Critical reflections

> **Disclaimer:** All summaries were created with the assistance of AI, based on course lecture transcripts and materials. They are not a substitute for taking the actual courses, but serve as study aids and reference materials.

## Contributing

Contributions are welcome. This is a community resource. Any improvement — new summaries, corrections, additional resources, or formatting enhancements — is appreciated.

### How to Contribute

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-contribution`)
3. Make your changes (see guidelines below)
4. Commit with a clear message
5. Push to your fork
6. Open a Pull Request describing your changes

### What You Can Contribute

- New course summaries (any course from the roadmap not yet covered)
- Additional resources (books, articles, videos, podcasts)
- Corrections (factual errors, typos, unclear explanations)
- Formatting improvements
- Supplementary business topics beyond the roadmap
- Real-world case studies
- Tool recommendations

### Contribution Guidelines

- Write in your own words – do not copy from lecture slides or transcripts
- Maintain the existing structure for new summaries
- Cite sources clearly with links or references
- Be objective – the "Reflections" section is for analysis, not personal evaluation
- Keep it concise – summaries are reference documents
- Use clear language and explain technical terms

### Review Process

Pull requests are reviewed within 1–2 weeks. Feedback may be requested before merging. Contributors are acknowledged in the repository's contributors list.

See [`CONTRIBUTING.md`](./CONTRIBUTING.md) for full details.
